# Code Binder Format

uDOS Developer uses a code binder to set up development work properly before implementation begins.

## Typical Contents

- **README** — overview of the work
- **dev-brief** — project intent, scope, constraints
- **project-mode** — FAMILY, FORK, or CONTRIBUTE
- **repo-context** — where the work lives
- **patch-plan** — planned changes or build path
- **tasks** — structured work items
- **contribution-notes** — upstream guidance where relevant
- **tool instructions** — handoff notes for IDEs and coding tools
