# Development Workflow

## Recommended Path

1. Start with **Agent Digital** to define the project clearly
2. Move into **uDOS Developer** when technical implementation is needed
3. Select a mode:
   - FAMILY
   - FORK
   - CONTRIBUTE
4. Run **SCAN** if the idea may already exist or can be adapted
5. Run **BRIEF** to create the code binder
6. Use **EXPORT** to hand the work into:
   - VS Code
   - Cursor
   - Codex
   - GitHub Copilot

## Publication

Development is private-first by default.

Use **RELEASE** only when the project is structured, documented, and intended for public publication.
